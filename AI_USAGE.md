# Spreetail Shared Expenses — AI Usage Document

This document records the utilization of AI coding assistants during the development of this repository.

---

## 1. Roles & Collaborative Workflow

The development was conducted in a pair-programming setup:
- **Human Developer**: Defined design parameters, schema constraints, business rules (especially date-gated active membership and CSV parser rules), reviewed generated code, and executed database runs.
- **AI Coding Assistant**: Structured the initial boilerplate, generated full model mappings, authored split strategies, implemented the 13 CSV anomaly checkers, and wrote template files.

---

## 2. Key Prompts and Directions

### 2.1 Project Initialization & Schema Design
- **Prompt**: Requested a clean folder structure containing `app`, `templates`, and `static`, alongside SQLAlchemy 2.0 async models using UUID primary keys and relationship linkages.
- **Outcome**: The AI constructed `app/models.py` with standard relational associations.

### 2.2 Split Arithmetic & Remainder Absorption
- **Prompt**: Directives to prevent floating-point issues, implement four splitting rules, and absorb remainders into the largest share.
- **Outcome**: Created `app/expenses/splits.py` using `Decimal` and `ROUND_HALF_UP`.

### 2.3 CSV Import Pipeline & Anomaly Engine
- **Prompt**: Provided CSV column specs and requested 13 distinct rules for duplicate detection, membership bounds validation, currency matching, Note keywords, and name-normalizing matchings.
- **Outcome**: Set up `app/importer/pipeline.py` and `app/importer/parser.py`.

### 2.4 User Authentication & Routing
- **Prompt**: Directives to set up custom bcrypt passwords and create signed cookie session flows with automatic 401 redirects to `/login` via FastAPI Exception Handlers.
- **Outcome**: Configured `app/auth/router.py`, `app/auth/service.py`, and `app/deps.py`.

---

## 3. Review & Verification

- Every module generated by the AI was tested programmatically.
- Automated python import verification scripts and integration test HTTP queries (`test_auth.py`) validated correctness before commits were made.
